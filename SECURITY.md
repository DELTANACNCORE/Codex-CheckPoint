# Security Policy

## Supported Versions

Security fixes are applied to the current `main` branch.

## Reporting a Vulnerability

Do not include credentials, session transcripts, private vault content, or exploit details in a public issue. Use GitHub private vulnerability reporting when it is enabled for this repository. Before publishing, enable that GitHub repository setting and configure a maintainer notification address.

If private reporting is unavailable, open a minimal public issue requesting a private contact channel without disclosing the vulnerability details.
