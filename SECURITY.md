# 安全策略 / Security Policy

## 支持版本 / Supported Versions

安全修复会应用到当前 `main` 分支。

Security fixes are applied to the current `main` branch.

## 漏洞报告 / Reporting a Vulnerability

不要在公开 Issue 中包含凭据、会话 transcript、私有 vault 内容或利用细节。仓库启用 GitHub 私密漏洞报告后，请使用该渠道；发布前应配置维护者通知地址。

Do not include credentials, session transcripts, private vault content, or exploit details in a public issue. Use GitHub private vulnerability reporting when it is enabled for this repository, and configure a maintainer notification address before publishing.

私密报告不可用时，只提交一条最小化公开 Issue 以请求私密联系渠道，不要披露漏洞细节。

If private reporting is unavailable, open a minimal public issue requesting a private contact channel without disclosing vulnerability details.
